Docker & Dockerfiles — Quick Reference

Practical examples, commands, and best practices for building small, secure container images.

What is a Dockerfile?

A Dockerfile is a text file that contains instructions for building a Docker image. Each instruction creates a new layer; smaller, focused steps produce smaller, cacheable builds.

Key keywords: FROM, RUN, COPY, ADD, WORKDIR, CMD, ENTRYPOINT, EXPOSE, ENV, USER

Example: Simple static site (nginx)

# Dockerfile.static
FROM nginx:stable-alpine
LABEL maintainer="you@example.com"
COPY ./public /usr/share/nginx/html:ro
EXPOSE 80
CMD ["nginx","-g","daemon off;"]

Place your site files in a public/ folder, then build: docker build -t my-static:latest -f Dockerfile.static .

Example: Multi-stage Node.js build (smaller runtime image)

# Dockerfile.node (multi-stage)
# Stage 1: build
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --silent
COPY . .
RUN npm run build

# Stage 2: runtime
FROM node:20-alpine
WORKDIR /app
# Use a non-root user (best practice)
RUN addgroup -S app && adduser -S app -G app
COPY --from=builder /app/dist ./dist
COPY package*.json ./
RUN npm ci --production --silent
USER app
ENV NODE_ENV=production
EXPOSE 3000
CMD ["node","dist/server.js"]

Build: docker build -t my-node-app:latest -f Dockerfile.node .

Example: Minimal Python app (use slim image)

# Dockerfile.python
FROM python:3.11-slim
WORKDIR /app
# Install only what we need
COPY pyproject.toml poetry.lock* /app/
RUN apt-get update && apt-get install -y --no-install-recommends build-essential \
 && pip install --upgrade pip setuptools \
 && pip install poetry \
 && poetry config virtualenvs.create false \
 && poetry install --no-dev --no-interaction \
 && apt-get purge -y build-essential && apt-get autoremove -y && rm -rf /var/lib/apt/lists/*
COPY . /app
EXPOSE 8000
CMD ["gunicorn", "app:app", "--bind", "0.0.0.0:8000"]

Tip: prefer slim/base images and remove build deps to keep image small.

Quick commands (cheat sheet)

docker build -t <name:tag> . — build image from Dockerfile in CWD
docker build -f Dockerfile.prod -t app:prod . — build with custom filename
docker run --rm -p 8080:80 app:latest — run and map ports
docker ps -a — list containers
docker image ls — list images
docker rm -f <container>, docker rmi <image> — remove
Pro tip: use --platform=linux/amd64 when building on M1/M2 machines if you need that platform.

Best practices & checklist

  • Use official base images and minimal variants (alpine, slim) when possible.
  • Remove build-time dependencies in multi-stage builds.
  • Pin versions for reproducible builds (e.g., node:20.4.0-alpine).
  • Don't run processes as root — create a user and use USER app.
  • Keep layers small and group related RUN commands to leverage cache.
  • Avoid secrets in Dockerfiles — use build args, environment variables, or secrets managers.
  • Scan your images (e.g., Trivy) before pushing to a registry.
